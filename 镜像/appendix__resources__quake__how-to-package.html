如何打包 Quake 地图/Mod

打包和发布可播放的 Quake 文件的一般规则和指南

本页介绍如何打包 Quake 地图/模组版本。

这些规则和指南基于大多数 Quake 引擎的工作方式以及过去几十年发展的一般社区礼仪和共识。

Quake诞生于1996年，其核心技术已有26年历史。就像任何 26 岁的人一样，现在改变他们身上任何不好或烦人的事情都为时已晚。我们必须弄清楚如何忍受它。

文件夹和文件名

  • 任何文件夹或文件名中都没有空格。

  • 全部小写字母。 （有些 Quake 引擎 + 操作系统区分大小写，有些则不区分大小写。）

  • 避免在开头使用数字或标点符号。

  • 使用常见的 .ZIP 压缩格式，不要使用不常见的深奥格式，例如需要特殊工具的 .7Z 或 .RAR。

封装类型

  • 地图包（简单）： 将所有内容放在 ZIP 的根目录中，没有子文件夹。

    • 适合简单的单一地图发布/您的第一张地图。

  • 模组包（复杂）： 将所有内容放入 mod 文件夹中，然后压缩整个 mod 文件夹。

    • 这样您就可以包含自定义资产和游戏代码。

    • 对于小版本来说，这样做也很常见，即使它们不是完整的模组。

地图包（简单）

如果您仅分发与地图相关的文件（.BSP / .LIT / .MAP / .TXT），那么可以使用最简单的打包方法：

要打包单个地图，请将所有文件放在 根 ZIP 文件的，ZIP 内没有文件夹或子文件夹。

$?/$

这是一种常见的结构，玩家知道他们必须自己将其解压到正确的地图文件夹中。我们假设这是 /Quake/id1/maps/ 除非另有说明。

模组包（复杂）

如果您使用自定义代码或资产，请将您的 Quake 版本打包为 mod：

要打包 mod，请将所有文件放在 ZIP 文件内的一个大“mod 文件夹”中。

但是，mod 文件夹必须 遵循特定的文件夹名称和结构...

$?/$

  • Mod 文件夹名称必须全部为小写字母数字且不含空格。

  • 地图文件（.BSP、.LIT） 必须位于 mod 子文件夹中 /地图/

    • 不要使用任何子文件夹。地图必须位于根目录 /maps/ 否则引擎无法找到它们。

  • 天空盒 (.TGA) 必须位于 mod 子文件夹中 /gfx/env/

    • 再次强调，不要使用任何其他子文件夹。

  • 声音 (.WAV) 必须进入 mod 子文件夹 /声音/

    • 此处鼓励添加其他子文件夹。使用唯一的声音子文件夹名称以避免文件冲突，例如 /sound/mymodname/

  • 型号 (.MDL) 必须进入子文件夹 /程序/

    • 鼓励添加其他子文件夹，例如声音。

  • 游戏代码（progs.DAT） 必须在 mod文件夹的根目录.

有些模组将所有内容都封装在一个 .PAK 文件以简化分发并避免此文件夹 soup. .PAK 文件就像 .ZIP 文件，Quake 引擎加载其中的文件，并且文件夹完好无损。有关 PAK 的更多信息，请参阅 QuakeWiki.org：PAK.

部分模组包

要打包部分 mod，请在根目录中包含多个 mod 子文件夹，但不要包含主 mod 文件夹。

包括有关解压缩位置的明确说明。

$?/$

  • 当您只有少量自定义资产时非常有用。

  • 主要用于作为现有大型模组（如奥术维度）的附加组件的地图。

    • 在您的版本中包含整个父模组是不切实际的。用户可能已经安装了自己的现有模组。

    • 然而，覆盖父模组的现有文件被认为是非常糟糕的礼仪，例如不包含 progs.dat 文件。

测试一下自己：Quake 包装示例

测试一下你自己。这是一个好例子还是坏例子？ 单击箭头以显示答案。

MyMod.zip

不好。文件名使用大写字母。

mymod.zip/MyMap.bsp

不好。文件名使用大写字母。

mymod.zip/mymap.bsp

好的。文件名全部小写，没有空格，BSP 文件位于 ZIP 的根目录下，没有子文件夹。如果作者只是发布地图文件而没有其他mod文件，那么这是可以接受的。

mymod.zip/mymodfolder/mymap.bsp

不好。它是一个 mod 文件夹，但 BSP 不在名为 /maps/ 的子​​文件夹中。

mymod.zip/mymodfolder/maps/mymap.bsp

好。所有文件和文件夹名称均为小写且不带空格，并且 BSP 位于 mod 文件夹 /mymodfolder/ 的子文件夹 /maps/ 中。

mymod.zip/mymodfolder/maps/mymod/mymap.bsp

不好。 BSP 位于名为“mymod”的地图子文件夹中，因此 Quake 引擎将无法检测到地图文件。

mymod.zip/mymodfolder/env/skybox_up.tga

不好。 TGA 文件位于 /env/ 中，但不在 /gfx/env/ 中。

mymod.zip/gfx/env/skybox_up.tga

好吧，也许吧。这是一种部分 mod 方法，只要用户知道将文件解压缩到哪个 mod 文件夹，它就可以工作。

mymod.zip/mymodfolder/gfx/env/skybox_up.tga

好。天空盒 TGA 位于正确的位置。

mymod.zip/mymodfolder/sounds/mymod/mysound.wav

不好。 “sounds”是不正确的文件夹名称。它必须是“声音”。是的，我知道。

mymod.zip/mymodfolder/gfx/env/skybox_up.tga + mymod.zip/mymap.bsp

不好。这是将一个文件打包到一个mod文件夹中，并将地图文件打包到根目录下。不要混合两种不同的方法。
