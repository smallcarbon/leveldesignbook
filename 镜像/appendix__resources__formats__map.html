MAP 文件格式

文件格式规范 + Quake .MAP 文件的示例解析器代码（“Valve”样式，mapversion 220）

这是一篇面向程序员的技术文件格式规范文章。

有关映射教程，请参阅 Quake 资源 相反。

.MAP 文件格式是为 Quake 1 (1996) 设计的 3D 文件格式。

它逐渐成为任何游戏引擎中基于笔刷的关卡的标准文件类型，部分原因是它的解析和编辑相对简单。这与 .OBJ 成为简单 3D 标准的方式类似，尽管它缺乏许多高级功能。

解析器

很有可能有人已经为您的游戏引擎/编程语言编写了通用的 .MAP 解析器。

  • C++（https://github.com/stefanha/map-files)

  • C# (https://github.com/Figglewatts/MapParse 或 https://github.com/LogicAndTrick/sledge-formats)

  • 铁锈（https://github.com/reslario/nomap 或 https://github.com/QodotPlugin/quarchitect)

对于特定于游戏引擎的 MAP 导入器和集成，请参阅 TrenchBroom.

文件格式

  • 基于大括号和换行符的文本

  • Z 轴向上（Quake 约定），右手

  • angle 值为 0-359 的欧拉度

    • -1表示“向上”

    • -2表示“向下”

    • 一些 Quake mods 将旋转实现为 3D 矢量，称为 m_angle

实体

.MAP 中的每个对象都必须是 实体，通用演员/游戏对象容器类型。

有两种类型的实体：

  • 点实体： 怪物、物品、灯光、固定尺寸/无尺寸的东西

  • 刷实体： 嵌入 3D 形状的点实体

    • 世界生成： 全局静态画笔实体，整个游戏世界的“根”

    • 世界画笔：绑定到 worldspawn 的画笔

实体数据和模板由 .烟气脱硫 文件 加载到关卡编辑器中。

画笔几何形状

一个 刷子 是由 4 个或更多平面定义的凸 3D 形状。

它不是由顶点和三角形组成的传统 3D 网格格式。 您需要额外的地图编译步骤来将笔刷数据处理为可用的 3D 网格。例如，Quake 地图使用名为 QBSP 的编辑器时命令行工具，该工具输出编译后的 .BSP 地图文件；同时，许多其他引擎的 .MAP 解析器在运行时生成此网格。

纹理/UV

Quake .MAP 文件格式有两个版本：原始标准版本和 Valve Software 更新的格式。

原始标准格式在存储纹理旋转时存在问题。给定 XYZ 中的 3D 笔刷平面法线，它会丢弃最长的轴并存储生成的 Vector2。但如果平面呈 45 度角，则最长轴现在不明确，地图编译器现在必须任意偏向某个轴； 浮点不精确 使得这种偏好更难以预测，通常会导致令人不愉快的纹理拉伸。

标准纹理坐标：

TEXTURE_NAME offsetX offsetY rotation scaleX scaleY

同时，更新的 Valve 格式 .MAP 具有 "mapversion" "220" 在 worldspawn 中设置的键值。 Valve 格式 220 通过将 UV 存储在 Vector3 空间中解决了 45 度模糊情况，这也使得一些有用的 UV 倾斜操作成为可能 TrenchBroom 以及。

Valve 格式纹理坐标：

TEXTURE_NAME [ ux uy uz offsetX ] [ vx vy vz offsetY ] rotation scaleX scaleY

我们通常建议使用 Valve 格式，除了向后兼容/将旧文件移植到更新的格式之外，没有理由使用标准格式。
