🏠白盒

如何通过体量、指标、寻路和试玩测试构建基本 3D 版本的关卡。

为什么要建立白盒？

一个 白盒 （也 块网格 或 灰盒) 是一个使用简单 3D 形状构建的 3D 草稿关卡，但没有任何细节或精美的艺术资源。

目标是制作、测试和调整关卡的基本形状。

在下图中，请注意白盒版本和最终发布版本之间的差异。一个形状可能一开始是一个灰色的块——然后经过几个月的时间 试玩测试 和 艺术传递，该块变成一堆桶......或者可能被完全删除。

白盒 支持实验.

删除或重建一些粗略的遮挡几何体是“便宜的”，但扔掉最终的 艺术过去了 工作“昂贵”且浪费。

当你对关卡结构没有信心时，最好保持它“便宜”，直到它准备变得“昂贵”。

您无法对设计文档或布局草图进行试玩测试， 但你可以测试白盒 并评估其 流动, 平衡, 遭遇战设计， 和 指标。这是设计阶段，您最终开始发现这些想法是否可行。白盒只是一个开始。

“……如果我们考虑到 [Neon White] 附带了超过 100 个关卡，团队的报废关卡数量达到了“双倍”，并且仅 Smackdown [关卡] 就做出了 50 多项更改，那么我们在整个游戏中将面临数千次调整和重新设计。[...] 三年后，Smackdown 完成了。”

——来自 霓虹灯白电平是如何制作的 作者：Blake Hester，游戏线人 #350

关键概念

屏蔽时，请考虑以下几个方面：

  • 体量 是形状所传达的一般体积和重量感。

    • 这个结构是厚/重，还是薄/轻？这是一个什么样的地方？

    • 风景 需要特别考虑。

    • 构图 目前在关卡设计文化中被过分强调。

  • 尺度指标 是关卡的一般比例、尺寸和比例。

    • 这个区域太大还是太小？玩家可以进入这个房间吗？

    • 有用测量的示例： DOOM 尺度, Quake 尺度

  • 寻路与引导 是玩家学习地图结构的导航过程。

    • 如何帮助玩家找到 关键路径 / 关卡退出？玩家是否感到太失落？

  • 试玩测试 是当您运行实验来查看关卡是否满足其设计目标时。

    • 大多数玩家都能完成该关卡吗？做 遭遇战 工作？是吗 平衡的?

    • 试玩测试确实很重要。这就是白盒的全部意义所在。

施工方法

游戏中常见的3D关卡构建方式有五种：

  • 原语： 排列简单的基本形状，如立方体和盒子。

  • 画笔/建模： 在关卡编辑器中构造 3D 形状。

  • 模块化套件： 将预制件连接在一起，就像乐高一样。

  • 雕刻： “绘制”有机 3D 形状，对于风景很有用。

  • 样条曲线： 创建按程序生成几何图形的数学曲线

基元

基元 是简单的 3D 形状，例如游戏编辑器中内置的默认灰色立方体和盒子。

移动/旋转/缩放形状。将它们排列在一起，相互堆叠，形成更复杂的形状组。

这是一种令人惊讶的强大白盒方法。您始终可以在任何 3D 引擎或工具中使用此方法，但您会遇到限制，无法屏蔽简单的四四方方的建筑物之外的任何内容。

关于使用原语进行阻塞的建议...

注意 指标, 建立规模是很棘手的。

做一个 坡道：制作一个长宽薄的立方体，然后旋转向下倾斜。使 楼梯：避免痛苦，只需制作坡道即可

做一个 门口： 两墙之间留有缝隙，没有门框。做一个 窗口： 制作类似门口的间隙，然后用齐腰高的短墙填充

画笔/建模

刷子 是在关卡编辑器中建模的简单低多边形 3D 形状。

这是 20 世纪 90 年代至 2000 年代整个行业的主要关卡构建方法。

我们强烈推荐这种方法，因为它提供了最强的控制力和手工制作的感觉，但不幸的是，大多数现代游戏引擎的画笔建模工具通常很差。在这里找出工作流程可能很棘手。

用刷子/低聚遮挡的建议...

使用一个 3D建模工具 强调低多边形几何。

  • 统一： 专业建造者 没问题，但是 佩刀CSG / 实时CSG 更加专注

  • 虚幻：使用 立方网格 + 建模工具 （或 网格工具)

  • 如果引擎可以使用 .MAP 或 .BSP 文件，请使用 TrenchBroom

使用一个 大粗网格尺寸 基于 指标，具体来说 播放器宽度。

  • 厄运 或 雷神之锤：32u 宽播放器 = 将网格设置为 32 或 64

  • Unity：1m宽的玩家=将网格设置为1或2

  • Unreal：60uu 宽播放器 = 将网格设置为 50 / 100 或 64 / 128

模块化套件

将套件中的预制建筑片段拼凑在一起 模块化零件.

然而，如果套件设计或配置不当，就会很难使用。如果您是初学者，您可能不应该尝试制作自己的工具包 - 只需下载一个即可。

有关规划、测量和构建模块化套件的更多信息，请参阅 模块化套件设计.

有关下载免费模块化原型套件的链接，请参阅 资源。

使用模块化套件进行白盒的建议...

配置网格并打开网格捕捉 （操作方法： 团结, 虚幻)

  • 根据模块大小使用大的粗网格

  • 例如如果墙体模块宽5m，则将网格设置为5

使用顶点捕捉 精确对齐模块（操作方法： 团结, 虚幻)

只是 下载预制套件。

雕刻

大多数现代发动机都有一个 雕刻工具 可以让您绘制平面并使其变形。

我们仅使用它来制作大而光滑的形状，例如地形和风景；这对建筑物等坚硬表面不利。

但如果景观是一个项目的核心，那么就无法避免雕刻。

有关雕刻和设计地形的更多信息，请参阅 景观地形.

用雕刻来遮挡的建议...

雕刻常常会让初学者分心。 只需单击一下即可制作一座山是如此有趣且充满力量，以至于您会忘记进行任何关卡设计。小心！不要让雕刻工具诱惑您！

使用大尺寸的画笔 首先关注大核心形状。

定义地平面 与 设置高度（统一） 或 展平目标（虚幻） 刷子

将斜坡雕刻成阶梯式露台 定义渐变，然后对其进行平滑。

避免侵蚀/自动发电机工具， 你需要控制设计。

样条曲线

样条曲线 是沿着路径生成几何图形的不可见曲线。

这些曲线也 非破坏性的;您可以在任意点调整样条线并重新生成其几何形状。

样条线非常适合遮挡道路或河流等弯曲的线性形状，它还可以自动使周围的景观变形。

也可以使用样条线生成非线性区域和建筑物，但这种类型的工作流程并不常见，通常需要自定义工具。

用样条线白盒的建议...

就像雕刻一样， 样条线通常会分散初学者的注意力。 使用样条线是如此有趣，以至于您会忘记进行任何关卡设计。不要让样条线诱惑您！

样条工具各不相同。 它们直到 2010 年代中期才变得普遍。 Unreal 具有出色的内置支持 蓝图样条线。 Unity 用户就没那么幸运了，必须尝试各种插件，例如 样条网格 或 编写您自己的样条线.

样条曲线并不神奇。 如果你想制作比道路、河流或桥梁更复杂的东西，通常需要定制工具和大量的艺术支持。

为什么不使用 Blender、Maya、Max 或 SketchUp 等 3D 艺术工具来遮盖？

人们争论这个. 我们的看法是：3D 艺术工具不允许您调整游戏玩法、碰撞、遭遇战或物品。试玩测试变得更慢且频率更低。

示例：在未命名的 AAA 游戏中，移动一块岩石需要一个长达一小时的多步骤过程：(1) 加载调试版本，(2) 查找哪个文件包含该对象，(3) 打开 Maya，(4) 编辑文件，(5) 重新编译游戏，(6) 再次加载调试版本以确认更改。

如何白盒

这是一个一般过程。如果您是初学者，或者正在开始一个新项目，请尝试做所有事情。您将逐渐个性化自己的流程和风格。

  1. 草图布局

  2. 添加地平面、比例尺图形、墙壁

  3. 试玩测试

  4. 再次发散、迭代和试玩测试

  5. 重复步骤4直至完成

1.S双桅帆船 布局

如果没有任何计划，可能很难白盒，尤其是对于初学者来说。即使是简单的涂鸦也能有很大帮助。

请看下面的布局草图。左边的草图强调了每个空间的规模，中间的草图侧重于区域之间的关系，而右边的草图是更详细的平面图。这些草图中的任何一个都可以工作，它们中的任何一个都可以帮助您规划一个草图。

就画点东西吧！

2. 添加地平面、比例尺图形和墙壁

创建地平面 （或一个大的扁平立方体）靠近 3D 空间的中心 (0, 0, 0)。

使用浅色网格原型纹理，以便您可以直观地估计尺寸和比例。该地板对象将帮助“接地”关卡几何体的其余部分，为其他所有物体提供水平线和上下文。

添加一个玩家人形比例人物 帮助建立规模。

如果您没有任何人形资产，请创建一个大致人形的块作为占位符。比例尺有助于玩家建立一致性和间隙。

有关常见走廊/墙壁尺寸和人形尺寸的更多信息，请参阅 尺度指标.

有关免费原型设计和遮盖纹理的链接，请参阅 资源.

建造一个墙段 大约是 比例人物高度的 150-200%.

如果您使用现代游戏引擎，尺寸不必非常精确——在这里建立比例感是最重要的。

如果您用 模块化套件，只需在图旁边放置一个标准墙模块即可。

用不同的颜色对墙壁进行纹理处理 从地平面。颜色和亮度为理解空间的组成以及这些不同的地板和墙壁平面如何相互关联提供了重要的背景。如果您担心颜色过多影响您的遮蔽，那么至少使用不同灰度的纹理。

复制并粘贴墙段 建立更多的空间，直到您至少拥有一个房间或走廊。有些工具具有特殊的“复制”或“克隆”命令：在 Hammer 中，按住 Shift 并拖动画笔；在 UE4 中，按住 Ctrl+Shift 并拖动墙壁对象。

交叉路口没问题。凌乱也没关系。

要制作门口或窗户，只需在墙段之间留出间隙，然后再将其填充即可。入口和走廊的宽度至少应是比例图的两倍。

3. 试玩测试

做一个简单的自我试玩测试： 在游戏引擎内，以完整的玩家重力、碰撞和速度在空间中行走。

白盒的目的是进行实验。要验证实验结果，必须测试原型。 不要只是在编辑器视图中飞来飞去， 这无助于你想象玩家的体验。

  • 体量. 关卡的形状有意义吗？是不是看着很混乱？

  • 尺度指标. 玩家是否能够穿过每扇门和通道？楼梯/坡道有用吗？玩家可以跳到他们应该跳的地方吗？

  • 流动感. 运动感觉是慢还是快、平滑还是复杂？哪种类型的运动最适合您的预期设计目标？

4. 发散和迭代

偏离原计划并响应试玩测试。

99% 的情况下，你的白盒都无法通过试玩测试。房间会感觉别扭，门口太窄，布局太混乱。但这正是我们白盒的原因！

想要重建大片白盒区吗？ 去吧。 想要删除整个房间吗？ 做吧。 需要将庭院分成两个较小的区域吗？ 你有许可。

对试玩测试和演练告诉您的内容保持开放的态度。

继续添加新比例的人物、墙壁和地板。

然后再次进行试玩测试。

5. 继续迭代

保持开放的心态并继续这个修改+试玩测试的循环。继续开发白盒。构建，然后在其中走动，然后修改，然后再次进行试玩测试......然后重复。这个设计过程称为 迭代，因为我们正在制作新的 迭代 它建立在先前版本的优势之上。

当您逐渐构建越来越多的关卡时，您可能会惊讶于它随着时间的推移而发生的变化。让地图给你带来惊喜。

常见的阻塞问题

  • （最常见）我的白盒感觉太小/太大。

    • 在施工过程中使用更多比例的人物。

    • 更频繁地进行试玩测试并更快地发现扩展问题。

    • 太大了？删除不需要的房间并压缩剩下的房间。

    • 太小了？扩展最外层的墙壁，然后扩展中心以填充新空间。

    • 或者删除所有内容并重建。这只是一个白盒。

  • 我很焦虑，盯着空荡荡的关卡编辑器屏幕，什么也没做。

    • 绘制布局图，然后尝试遵循该计划。

    • 只需白盒一间简单的房间即可。您稍后可以随时将其删除。重点是摆脱空白页。在绘画中，这就是“激活画布”；用书面形式来说，这是一个 “糟糕的初稿。”

    • 做点别的事，然后再回到白盒状态。

白盒示例

肮脏炸弹的“城堡”（溅射伤害）

Splash Damage 关卡设计师 Anthony "MassE" Massey 为竞技性多人团队射击游戏 Dirty Bomb 设计了地图 Castle。梅西从蜿蜒的中世纪街道和现实生活中的伦敦塔类型中汲取灵感，并概念化了最初的布局图（上图左）。但请注意生成的遮挡（上方、中间）与曲线较少、90 度角较多和区域较大的布局有何不同。布局只是一个初步指南。由于这是一张多人射击游戏地图，设计团队对白盒进行了试玩测试，重点关注移动和战斗 指标:

“地图的比例始终是最难做好的方面。绘制主要路径并测量这些距离非常重要；我们正在寻找从生成到目标的 8 到 12 秒之间的任何时间。[...] 任何比这更长的时间，玩家重生时都会感到沮丧并不得不跑回来。另一方面，如果时间较短，则可能会导致地图对于 8v8 比赛来说太小，并可能导致游戏玩法混乱。[...] 这个阶段对于地图开发至关重要，但我们的团队遵循一个简单的经验法则：如果感觉很长，那就太长了。”

“下一步是考虑[玩家类别]。我们必须考虑战斗范围，以使我们的整个武器库发挥作用，[...]同时还要确保战斗空间的多样性，以使[不同的能力]可行（例如[空袭能力]）。实际上，这意味着考虑室外和室内区域之间的比例，确保[具有室外能力的角色]可行。”

——来自 “Blocktober：脏弹”Splash Damage 博客文章

《Apex 英雄》的“世界边缘”（重生）

Respawn 关卡设计师 Rodney Reece 为第一人称大逃杀游戏《Apex Legends》的大型多人竞技地图“World's Edge”构建了白盒。请注意地图如何从遮挡（左）到美术处理（右）发生巨大变化，不仅在主题上，而且在布局和构图上。

“最初的想法是地图下雪。但艺术想要将绿色带入地图，在一次会议中，罗伯特·陶布提出如果震中（当时称为冰冻爆炸）导致下雪怎么办。

成为一名优秀设计师的关键是对新想法保持灵活。例如，艺术团队提出了圆顶。在我的块网格中，它是一座火山。他们提出了这个想法，我进行了调整以融入这个想法。我必须为它创建一个新的布局，但它使它变得更好。

但其他时候，某些[兴趣点]从一开始就很有趣。在这种情况下，我对可以改变的事情更加固执。分拣工厂就是一个很好的例子。确定布局方面什么是宝贵的以及什么是灵活的很重要。因为这需要一个团队！”

反对白盒？

对于某些项目，传统的白盒可能不太有帮助。体验可能更少取决于空间设计，而更多取决于 美术处理。白盒无法验证依赖于艺术或其他资产的概念。

“Firewatch”的垂直切片（Campo Santo）

对于第一人称叙事探索游戏 Firewatch (2016)，开发商 Campo Santo 希望专注于行走和说话等机制；游戏概念的主要吸引力是观看艺术传递的风景并聆听配音对话。按照典型的最佳实践，他们首先构建了一个白盒来测试这些机制的可行性。然而，白盒并没有帮助他们回答任何有关玩家体验的问题，因为游戏节奏从根本上来说是一个叙事设计和环境美术问题，而不是一个关卡设计问题。传统的白盒流程不起作用。

根据环境美术家 Jane Ng 的说法，目前尚不清楚《看火人》是否会作为一种体验“发挥作用”，直到他们 跳过了白盒过程 相反，完成了一个垂直切片原型，具有艺术通过的环境和接近最终的对话。

《无题鹅游戏》外景拍摄（House House）

对于自上而下的第三人称潜行益智游戏《Untitled Goose Game》，开发商 House House 试图以 3D 方式创建一个真实的英国村庄。关卡设计师杰克·斯特拉瑟（Jake Strasser）从未去过英国，因此在最初的关卡中偶尔使用了照片参考。然而，这种白盒优先的方法要求斯特拉瑟用他的想象力来填补空白——而且因为他没有英国人的想象力，这些空白常常以微妙的方式让人感觉难以置信或不真实，与他们的意图背道而驰。传统的白盒流程不起作用。

因此，斯特拉瑟尝试了一种 研究-第一种“位置侦察”方法。他对英国各个村庄进行了详尽的虚拟游览，并从谷歌街景中拍摄了大量照片。在游戏中重建了几条村庄街道后，斯特拉瑟最终确定了适合游戏需求的结构类型，并根据 奥福德村泵街。在这里，白盒不再是建造一个空间，而是更多地发现一个空间。

一个 白盒 是一个简单但可玩的“草稿”关卡，细节极少。我们保持简单和块状，以便我们可以轻松更改它。

白盒方法：

  1. 素描一个 布局，即使是一个简单的

  2. 添加一个 地平面， 比例尺数字， 和 墙壁.

  3. 试玩测试 立即；逛逛感觉还好吗？你现在需要知道。

  4. 发散 从你原来的计划来看， 再次迭代并进行试玩测试

  5. 重复步骤 4。

最常见的问题是规模，特别是对于初学者。在游戏中测试白盒是了解它是否感觉太大或太小的最好方法。
