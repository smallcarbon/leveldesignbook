🌈发布

如何记录和分发关卡

一旦你制作了原型、进行了试玩测试并完善了你的关卡，你可能应该 发布 公开它。

您不必 100% 完成项目才能发布它。您可以发布 alpha/beta 版本，甚至只是一个简短的演示。不要等到某件事完全完美才去做，因为很可能没有什么看起来足够完美。许多设计师和开发人员都会发布未完成的项目 - 有些人甚至将其出售，并且他们的玩家喜欢它！

在此页面上，我们将介绍公开展示自定义关卡、地图包、模组或独立项目的一般最佳实践和规范。

如何记录关卡

要向其他人展示您的项目，您至少需要以下内容：

  • 3+ 游戏内截图 的 主要领域 （没有随意的走廊或壁橱）

    • 至少 1920x1080 分辨率

    • 高品质 .JPG（~80%+ 品质）

    • 如果可能的话，禁用 HUD/UI 元素和敌人 AI

    • 不要拍摄漂亮的墙壁或道具；相反，你的目标是给人一个总体印象 布局, 动线、心情、以及 空间构图

  • 2-10分钟的游戏视频 行动中的水平

    • 30 fps 时至少 1280x720 分辨率

    • 省略任何启动画面、介绍画面、预告片音乐等。

    • “正常”玩，展示“普通”玩家如何体验关卡

    • 记录与 开放广播工作室 (OBS)，上传到 YouTube 或 Vimeo

      • 剪辑素材可能很好，但并不一定重要

      • 可选：将您的个人音频设计评论叠加为可选音轨，向老师或未来雇主展示流利程度和沟通技巧

  • 50-100字描述 关于该项目

    • 给它起一个简短好记的名字

    • 单人游戏、多人游戏？核心主题、情感、机制？

    • 范围是什么？这个项目有多少层、有多大？

    • 项目学分，谁参与了这个工作？资产从哪里来？

  • 下载链接

    • 对于游戏玩家，您必须包含 Windows 兼容版本

    • 如果它是带有 Steam 创意工坊等的游戏的自定义关卡/模组，请在那里链接

    • 否则，您可以将文件托管在免费服务上，例如 痒痒io, 游戏震动, 或 模块化数据库

      • 使用.ZIP...避免使用 .EXE 安装程序或非传统格式（如 .RAR 或 .7Z），这些格式不易访问且需要特殊软件

作品集的额外材料

如果您正在准备关卡设计作品集，希望获得关卡设计师的职位，那么您可能需要准备一些额外的文档：

  • 带标签的自上而下地图概览

    • 在编辑器工具的 3D 视图中，向上飞翔并向下看，截取屏幕截图，然后在 2D 工具中打开它并添加文本标签/标记

      • 要在 Windows 上截取屏幕截图：按 Windows Key + Shift + S，然后拖动一个矩形

      • 在 MacOS 上截取屏幕截图：按 Command + Shift + 4，然后拖动一个矩形

    • 对于单人游戏：沿主动线绘制或编号各个点

      • 对于战斗密集的关卡，显示敌人的布局和战略位置

      • 对于谜题，显示解决谜题的线索/步骤的顺序

    • 对于多人游戏：标记玩家生成点、捕获点等。

  • 处理图像

    • 显示布局图、草图截图...“展示您的作品”

    • 但不要上传太多，只需上传一些即可证明您了解该流程

    • 如果你的布局图难以理解/混乱，最好不要显示它

  • 100 字的开发日志/分析

    • 回顾过去，这个项目的主要优点和缺点是什么？

    • 随着时间的推移，项目发生了怎样的变化？有什么有趣的试玩测试或故事吗？

    • 其他游戏的任何关卡对你的方法有启发吗？为什么是那些？

    • 表明你可以思考和沟通，但不要写太多

    • 任何新闻报道或奖项的链接

除了上述内容之外，请确保包含屏幕截图、游戏视频和下载链接。

除非您进入最后一轮招聘，否则未来的雇主很可能不会下载并玩您的关卡 - 但下载链接仍然很重要，因为它表明您知道如何完成和发布某些内容。如果下载页面充满用户反馈和高社交媒体标准尺度规范，看起来会更好。虽然雇主自己可能达不到你的水平，但他们肯定关心其他人是否达到了你的水平。

项目越受欢迎/商业化，您需要的文档就越少。 AAA 级设计师只需几张截图就可以了，因为每个人都已经知道他们的作品，而大型工作室经常白盒员工发布任何未经官方批准的内容。然而，如果您是一名默默无闻的学生/新手设计师，正在寻求第一份游戏行业工作，那么不幸的是，没有人会对您或您的工作如此熟悉，您将不得不更加努力地让他们相信您的能力。

不要在您的作品集中包含 Minecraft 版本、自定义 《堡垒之夜》 地图或 Roblox 项目。尽管这些游戏中出现了一些最有趣的关卡设计实践，但不幸的是，业界普遍认为这些工具不专业。

如何发布项目

每个游戏和平台都有自己的社区规范和发布项目的惯例。一些 mod 社区可能更喜欢 Steam Workshop 之类的自动安装功能，而对于独立项目，您可能需要自己将文件托管在 itch.io 等免费游戏平台上。

大多数关卡/游戏的基本发布流程通常如下所示：

  1. 收集媒体： 截取屏幕截图、录制并上传游戏视频、准备文字说明和制作人员名单

  2. 套餐 .ZIP 文件中的可播放项目文件以及 上传 某处

    • 包含该关卡可能需要的任何其他文件，并且所有子文件夹完好无损

    • 包括一个 readme.txt 包含简要说明和解压缩文件位置的说明的文件

    • 在 Windows 上制作 .ZIP： right-click on folder > Send To > Compressed folder

    • 在 MacOS 上制作 .ZIP： right-click on folder > Compress items

  3. 发布 发布

    • 创建一个项目页面（在 itch.io、您的个人网站、Steam 创意工坊、论坛等上），其中包含文本描述、制作人员名单、屏幕截图、游戏视频嵌入和下载链接

    • 在社交媒体上宣传项目页面，恳求人们玩并分享它

      • Twitter（游戏行业/媒体记者青睐的社交平台）

      • Discord（游戏玩家的主要社区平台，尤其是模组社区）

      • Instagram（普通人/非游戏艺术家的流行视觉平台）

      • ArtStation（游戏艺术家的主要视觉平台，环境美术家必不可少）

如果您要出售您的项目，那么公开发布实际上只是项目生命周期的一半。将会有大量的反馈、错误报告、修复和补丁需要解决。玩家将期待维护和更新。如果你管理得好，你也许能够一次又一次地重新启动游戏，并随着时间的推移培养健康的“长尾”收入……但如果你正在制作商业版本，希望你不会从The Level Design Book 中文版中寻求商业建议，所以我们就到此为止。

如何摆脱绝望

即使你完成了你的项目，也很可能没有人会玩它。 99%的关卡有十几名玩家，一百次下载，最多一千次浏览量。

缺乏大规模的公众反应可能会让人感到非常失望、疏远和沮丧。但这种经历很常见。你并不孤单。如果你没有获得巨大的、惊天动地的成功，那么这并不意味着你是失败的——它使你成为一个人。

记住：

  • 为小社区和受众制作地图、模组和游戏是可以的……您可以为一个受众创建；这些被称为礼物

  • 可以从事小型项目，也可以从事非商业项目

  • 尝试某件事然后永远完成它是可以的

  • 你不需要在一家著名的大型游戏工作室工作才能成为一名出色的关卡设计师

  • 关卡设计是一门艺术，需要磨练其内在美

  • 每个创意领域都有一个不变的事实，那就是对创意人员的需求 社区 理解其中的起伏……如果感到痛苦，请与某人交谈

  • 你不仅仅是一个项目！继续前进！
