FGD 文件格式

《雷神之锤》 时代关卡编辑器的实体定义文件/映射辅助

烟气脱硫 是一种常见的文本文件格式，在多个 《雷神之锤》 时代的关卡编辑器工具中使用（例如 TrenchBroom) 来定义实体。

安 实体 基本上是任何非静态世界几何体的游戏对象——NPC、物品、灯光、声音、门、玩家生成点等。任何可以移动、改变或反应的东西。

请注意，FGD 只是添加模板和文档的编辑辅助工具。 它本身不实现任何引擎内游戏功能。 它只是帮助关卡设计师理解和配置对象。从技术上讲，您不需要 FGD 来为游戏绘制地图；理论上，您可以简单地手动记住并验证所有关卡脚本数据。

有关常见游戏对象类型和行为的更多信息，请参阅 脚本编写.

本页假定您对常见的 《雷神之锤》 映射术语有一定的了解。

历史

FGD 代表“Forge Game Data”，是《半条命 2》关卡编辑器 Hammer 成为《半条命 1》编辑器 Worldcraft 时的遗留物，在此之前是名为 Forge 的 《雷神之锤》 工具。

由于多个工作室、引擎和编辑器工具都使用了 FGD，因此 FGD 有几种不同的“风格”。例如，Valve 在 Source Engine 中添加了语法来支持其独特的实体输入/输出脚本系统。 目前尚无正式的 FGD 文件规范标准。

本页涵盖了 FGD 中使用的核心语法 TrenchBroom 适用于《雷神之锤 1》/《半条命》引擎。

语法（基本）

FGD 是一种基于文本的格式。我们建议使用 UTF-8 编码以支持更多语言。 TrenchBroom 需要 UTF-8，不带 物料清单.

每个实体条目都遵循以下模式：

// 注释以“//”开头
@EntityType EntitySettings() = EntityClassName :“编辑器内帮助”
[
    propertyKey(propertyType) : "编辑器内名称" : defaultValue : "编辑器内帮助"
]

实体类型和设置（基本）

FGD 实体分为三种核心类型： 点, 固体， 和 基地.

  • @PointClass 开始点实体定义，例如 NPC 或灯光

    • 尺寸(minX minY minZ, maxX maxY maxZ) 是点实体的轴对齐边界框 (AABB) 大小，定义为两个 3D 向量：最小范围（最小角）和最大范围（最大角）

    • 型号(path, skin, frame, scale) 显示实体的 3D 模型。这可能会变得有点复杂，请参阅 实体的显示模型.

  • @SolidClass 开始画笔实体定义，如 worldspawn 或 func_wall

  • @基类 开始实体基础模板；您可以将实体设置为“继承”公共基础。

    • 基地(BaseClassName1, BaseClassName2, ... ) 设置一个实体以包含列出的基类...您还可以设置一个基类以包含一个基类，因此此功能可能非常强大/有用，但要注意递归

例如，如果游戏中的所有怪物都有生命值，那么您可以创建一个具有生命值属性的 BaseClass Monster，然后将所有怪物类型设置为包含该怪物基础：

$?/$

实体属性（基本）

实体属性是遵循以下一般模式的类型化键值对：

键名(类型) : "In-Editor Display Name" : 默认值 : "In-Editor Help"

  • 按键名称： 像变量名一样，通常是小写字符串

    • 有时开头带有下划线，表示它是编译器时属性，不会出现在游戏中

  • 类型： 有四 (4) 种基本属性类型： string, integer, choices （多项选择整数 枚举），以及 flags （位掩码）。

    • 旗帜 是一组布尔（真/假）值，通过数学的魔力将它们存储为单个数字（位掩码)

    • 在大多数关卡编辑器中，一个实体只能有一个 flags 属性，并且必须调用它 “产卵标志”

  • 价值： 无论关卡设计师在关卡编辑器中输入什么内容

实体的显示模型

如果 FGD 文件指定，TrenchBroom 可以显示点实体的 3D 模型。这对于放置 NPC 或道具非常有用，其中预览模型大小/旋转/外观非常有用。

  • TB 使用资产导入器 (AssImp)，它对大多数 3D 格式（.OBJ、.FBX 等）具有基本支持。

  • 对于某些模型格式（例如 《雷神之锤》 .MDL 或 《半条命》 .MDL），您还可以根据实体属性交换纹理或预览动画帧。使用方法见下文。

记住： 模型（） 仅限 TrenchBroom，不一定适用于其他编辑器。
